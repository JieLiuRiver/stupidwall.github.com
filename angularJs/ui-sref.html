 /*  	通过ui-sref指令构建链接
 /*===============================================*/
	$stateProvider.state('thread.show',{
		url : '/:id/show?title&poster',
		templateUrl : 'controllers/thread/show.html',
		controller  : 'ThreadShowCtrl as vm'
	})

	angular.module('com.ngnice.app')
		.controller('ThreadShowCtrl',
		function ThreadShowCtrl($stateParams){
		var vm = this;
		vm.id = $stateParams.id;
		vm.title = $stateParams.title;
		vm.poster = $stateParams.poster;
	});

	<a ui-sref="thread.show({ id :node.id, title:node.title, poster:node.poster })">查看</a>

				
			
				
	代码说明：
	1、
	首先新增一个路由定义：
	其中声明来2个路由参数，
	一个是行内的id参数，一个是query参数title

	2、
	接下来是再控制器中使用：
	这里的关键是$stateParams服务
	凡是从路由传过来的参数，都可以通过这个服务访问到
	但是有一个坑：
	只有在路由定义中声明过的参数才能使用
	比如：
	这里声明来2个query参数，title、 poster
	如果访问的url是:
	/thread/1/show?title=abc&poster=wzc&dateCreated=2015-03-12
	title poster接收正常
	dateCreated参数为空，因为前面没有声明它

	3、
	其中ui-sref是angular-ui-router提供的指令，
	参数是路由的名字
	括号中的参数是路由参数组成的对象

	如果说，使用系统内置的ngRoute作为路由库
	那么就需要自己拼接ng-href
	可读性、灵活性比较差
